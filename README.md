## Business Workflow

The workflow I chose is summarizing meetings into action items.
The user is a business professional or team member who attends meetings and needs to quickly understand what actions need to be taken after the meeting.
The input to the system is a set of meeting notes or a meeting transcript, which may be unstructured and contain extra information.
The output of the system is a clear and structured list of action items, including tasks, responsible persons, and deadlines if available.
This task is valuable because meetings often produce a lot of information, but it is difficult to quickly identify what needs to be done. Automating this process can save time and improve team efficiency.
